# Grid-Based Level Format

This directory contains level definitions in a simple grid-based ASCII format that makes it easy to design levels visually.

## Format

Each level is defined in a `.grid.txt` file where:
- Each line represents a row (Z coordinate)
- Each character represents a column (X coordinate)
- The grid is centered at (0, 0) in world space
- Each cell = 2 units in world space (configurable)
- Grid supports up to 200x200 cells (400x400 units in world space)

## Symbols

- `.` = Empty space
- `S` = Start position (ball start)
- `H` = Hole
- `B` = Bumper
- `F` = Fan
- `M` = Magnetic Field
- `P` = Power-up
- `W` = Wall (custom wall)
- `R` = Rectangular hole (hazard)

## Converting to JavaScript

To convert a grid file to a JavaScript level file:

1. Open the grid file in a text editor
2. Copy the grid content (without comments)
3. In browser console or Node.js, use:

```javascript
import { convertGridToLevelCode } from './js/grid-to-level.js';

const gridString = `
.........................
.........................
S......................H
.........................
.........................
`;

const code = convertGridToLevelCode(gridString, 'level5');
console.log(code);
```

4. Copy the generated code into your level file (e.g., `js/levels/level5.js`)

## Tips

- Use a monospace font to edit grid files
- Comment lines (starting with `//`) are ignored
- Grid size determines course dimensions automatically
- You can adjust `cellSize` in options to change scale
